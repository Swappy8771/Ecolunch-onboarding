Must-Show KPIs

  • Open EcoLoop tickets
  • Blocked go-live files (count)

  • Go-live Monitor

⚠ "Open Support Access Session" — this is an audited support access session, not a normal user switch. Do not label it "Switch to Caterer Portal." Log the session start/end, actor, and reason in audit_logs.

Standard Contract Types
  • Master Service Agreement (MSA)
  • Non-Disclosure Agreement (NDA)
  • Data Processing Agreement (DPA)
  • EcoLunch Platform Terms
  • Food Safety Compliance Annex
  • Module Annexes (per activated module)
  • Fee Schedule Annexes

Signing Workflow — Dropbox Sign API

Admin selects caterer → picks standard contract template (template_id stored in config)

Admin confirms merge field values: client_name legal_name monthly_rate start_date module_name fee_percentage signatory_email

Backend calls Dropbox Sign API: signature_request/send_with_template with template_id + signers + custom_fields

Dropbox Sign handles: merge, PDF generation, email to signer, signing workflow, audit trail

Signer receives email → views → signs inside Dropbox Sign

Dropbox Sign fires webhook → backend receives signature_request_all_signed

Backend downloads signed PDF + audit trail from Dropbox Sign

Backend uploads signed files to Dropbox Storage at /09_Contracts_Signatures/

DB stores only: dropbox_file_id dropbox_file_path signature_request_id — never binary PDF

Contract status updated → Document Vault updated → Go-live checklist re-evaluated → audit created

⚡ Activation rule: A signed contract alone does not activate the caterer. Signed → contract requirement marked complete → Go-live checklist re-evaluated → caterer becomes Active only when ALL checklist items pass.

07

Go-live Monitor

Verifies whether each caterer has completed every required onboarding step and is eligible for activation. Activation cannot occur unless all 11 checklist items are complete. The system evaluates this automatically after each relevant change.

11 Required Checklist Items

  • account_created — caterer user account exists
  • profile_validated — all required profile fields approved
  • banking_validated — banking info approved
  • establishments_confirmed — at least one entity confirmed
  • menus_validated — at least one menu validated
  • documents_approved — required docs received + approved
  • contracts_signed — all required contracts signed
  • modules_configured — modules selected + configured
  • pricing_configured — pricing set for all active modules
  • corrections_closed — no open corrections
  • ecoloop_blockers_closed — no blocking EcoLoop tickets
Monitor Shows per Caterer
  • Overall readiness %
  • Completed vs. blocking steps
  • Missing required documents
  • Unsigned contracts
  • Open validations / corrections
  • Modules / pricing not configured
  • Menus not validated
  • Establishments not confirmed
  • Ready / Not Ready status
Admin Actions
  • View blockers detail
  • Open blocking section directly
  • Send client reminder
  • Send via EcoLoop
  • Validate Go-live (triggers activation)
  • Block Go-live (manual override)
  • View audit trail
✓ After Go-live approval: approved data activates in caterer operational portal · caterer status → Active · modules become effective · configured pricing/rates become effective · full audit log created.

EcoLoop Onboarding

The communication and follow-up layer for onboarding. Functions as a support ticket system + threaded messaging between EcoLunch admin and caterer. Every correction request, validation follow-up, and go-live blocker can be linked to a ticket.

Ticket Contains
  • Caterer identifier
  • Subject + status + priority
  • Assigned admin
  • Threaded conversation (admin ↔ caterer)
  • Internal notes (admin-only)
  • System action log
  • Linked validation item(s)
  • Linked document(s)
  • Linked contract
  • Linked Smart Import item
  • Linked go-live blocker
Actions
  • Create ticket (manual or auto from correction)
  • Send message (to caterer)
  • Add internal note (admin-only)
  • Link to validation item
  • Link to document
  • Link to contract
  • Link to Smart Import item
  • Link to go-live blocker
  • Close / Reassign / Change priority

Message Types

admin_to_client client_to_admin internal_note system_action

Smart Import Engine

Smart Import and Smart Upload are the same engine. It is a contextual intelligent import used in any onboarding section where a document or structured file can be uploaded to auto-fill fields. It is never a standalone sidebar item.

⚡ Critical rule: No extracted data may be applied automatically without human review. Every field must be confirmed, edited, or rejected by a user.

Where It Appears

Profile Banks & Banking Information My Clients / Establishments Menus & Packages Document Vault Contract Management (where applicable)

14-Step Workflow

User is inside a specific onboarding section → clicks Smart Import

User uploads document or structured file (PDF, Excel, CSV, Acomba, QuickBooks export)

File stored in Dropbox Storage under caterer's vault

System detects document type (legal doc, void cheque, menu file, school list, etc.)

System reads and parses file content

System extracts candidate field values

Fuzzy matching against EcoLunch target fields

Human review screen displayed — all extracted fields shown with confidence scores

User reviews each field: Confirm Edit Reject

Only confirmed values applied to EcoLunch fields

Document classified in caterer's Document Vault (correct category)

Validation items created where required (→ Validation Center)

Full audit trail created for every field action

smart_import_job marked as applied

Import Statuses

Job Statuses
Uploaded · Processing · Extraction Complete Needs Review · Partially Mapped · Mapped Confirmed · Applied Rejected · Correction Requested · Archived
Field Statuses
Detected · Mapped · Low Confidence Needs Review Confirmed · Edited · Applied Rejected
