How the admin approves banking (across two databases)

The outbox keeps PostgreSQL and MongoDB in sync

The caterer's banking lives in PostgreSQL, but the admin's review queue and the status live in MongoDB. When the caterer submits banking, a transactional outbox reliably creates the validation_item the admin reviews and sets banking_status = under_review. The admin then approves it in the Validation Center — and the Go-live checklist's banking_validated recomputes automatically.

Admin rules that never bend

›Sees everything. Admin queries are not scoped to one caterer.
›Signing ≠ activation. A signed contract only ticks one checklist item.
›Go-live is 100% or nothing — all 11 items, and only an admin approves.
›Support sessions are audited — viewing a caterer's portal logs start & end.
›Internal notes stay internal — never returned to the caterer.
